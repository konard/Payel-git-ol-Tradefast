# Arbitrage Strategy

## Суть стратегии
Arbitrage (арбитраж) — это одновременная покупка и продажа одного и того же актива (или связанных активов) на разных рынках/биржах с целью получения прибыли от разницы цен без принятия направленного риска.

## Где она работает
- Между разными криптобиржами (spot arbitrage)
- Между spot и фьючерсами (basis trading)
- Между связанными активами (pairs arbitrage, ETF vs underlying)
- На разных платформах с разной ликвидностью

## Главная идея
Один и тот же актив не может долго стоить по-разному на разных площадках. Арбитражёр зарабатывает на временных неэффективностях, покупая дешевле и продавая дороже практически одновременно.

## Почему стратегия популярна
- Теоретически низкий риск (market neutral)
- Хорошо поддаётся автоматизации
- Может работать 24/7 в крипте
- Меньше эмоциональной нагрузки

## Основные компоненты
- Мониторинг цен в реальном времени на разных площадках
- Быстрое исполнение ордеров
- Учёт комиссий, slippage и funding rate
- Управление капиталом и балансами на разных биржах
- Механизмы хеджирования

## Базовые таймфреймы
Исполнение происходит на очень коротких интервалах (секунды — минуты). Анализ — постоянный.

## Логика входа
- Обнаружение расхождения цен выше порога (с учётом комиссий)
- Одновременное открытие длинной и короткой позиции
- Быстрое закрытие при возвращении спреда к норме

## Где стратегия ломается
- При высоких комиссиях и slippage
- При проблемах с выводом/вводом средств
- Во время сильной волатильности и отключения API
- При изменении правил бирж (например, запрет short)

## Типичные ошибки
- Игнорирование всех издержек
- Слишком большой размер позиции
- Отсутствие плана на случай проблем с одной из бирж
- Ручное исполнение вместо автоматизации

## Риск-менеджмент
- Строгий лимит на максимальный размер арбитражной позиции
- Мониторинг доступности средств на всех площадках
- Автоматическое закрытие при превышении времени удержания
- Диверсификация по разным парам и биржам

## Психология
Арбитраж кажется "безрисковым", но на практике есть много скрытых рисков. Главные ловушки:
- Переоценка скорости исполнения
- Игнорирование операционных рисков
- Жадность — слишком большой объём

## Что автоматизировать в Lostfast
- Real-time мониторинг спредов между биржами
- Автоматическое исполнение арбитражных сделок
- Учёт всех комиссий и funding в расчёте прибыльности
- Мониторинг балансов и рисков на разных площадках
- Логирование всех арбитражных операций

## Мини-шаблон правил
- Торговать только когда спред покрывает все издержки + маржу безопасности
- Автоматическое закрытие при недоступности одной из бирж
- Жёсткие лимиты на размер позиции
- Регулярный аудит всех арбитражных сделок
